# Skrypt sesji: pacjent lub opiekun

Status: neutralny skrypt M5. Uzywaj tylko z fikcyjnymi case studies.

## Cel rozmowy

Sprawdzamy, czy pacjent lub opiekun rozumie, jak przygotowac kontekst przed wizyta: dokumenty, leki, pytania, braki danych i zakres dostepu.

Nie zbieramy realnych danych medycznych. Nie prosimy o historie zdrowia uczestnika.

## Otwarcie

"Pokaze fikcyjne demo. Prosze uzywac go tak, jakby trzeba bylo przygotowac sie do wizyty. Jezeli cos wyglada jak instrukcja medyczna albo jest niejasne, prosze powiedziec to od razu."

## Zadanie glowne

1. Pokaz widok pacjenta lub opiekuna.
2. Popros: "Znajdz najblizszy kontakt medyczny albo najwazniejszy element do przygotowania".
3. Popros o wskazanie dokumentow lub wynikow do zabrania.
4. Popros o wskazanie listy lekow i informacji do potwierdzenia.
5. Popros o znalezienie jednego pytania do lekarza.
6. Zapytaj, kto ma dostep do jakich obszarow danych.

## Pytania po zadaniu

- Co bylo jasne od razu?
- Gdzie trzeba bylo zgadywac?
- Czy widac, ze dane sa fikcyjne?
- Czy widac roznice miedzy tym, co wiadomo, a tym, co trzeba potwierdzic?
- Czy system pokazuje nastepny krok organizacyjny?
- Czy jakikolwiek tekst brzmial jak polecenie medyczne?

## Metryki szkody

- H-001: czy uczestnik odczytal widok jako rozstrzygniecie kliniczne?
- H-008: czy kolor lub flaga wygladaly jak ocena kolejnosci kontaktu?
- H-009: czy tekst mogl sprawic wrazenie, ze rozmowa z lekarzem jest mniej potrzebna?
