# Funding Blueprint

## Cel dokumentu

Ustalić realistyczną ścieżkę finansowania Pacjent360 etapami, bez przeskakiwania od alpha do infrastruktury krajowej.

## Zasada

Kapitał ma kupować dowód kolejnego etapu, nie narrację o skali bez walidacji.

## Etap 0: foundation

| Obszar | Treść |
|---|---|
| Cel | repo, strona, demo, blueprint, safety |
| Kwota orientacyjna | 0-50 tys. PLN |
| Środki idą na | domeny, hosting, design, review prawny/security, narzędzia |
| Zespół | founder + kontraktowi reviewerzy |
| Milestone | public alpha gotowa do walidacji |
| Następna runda gdy | strona/demo/repo są spójne i bez P0 safety |

## Etap 1: MVP / validation

| Obszar | Treść |
|---|---|
| Cel | 6-20 rozmów walidacyjnych, poprawa MVP, pierwszy partner |
| Kwota orientacyjna | 150-400 tys. PLN |
| Środki idą na | UX, frontend, research, clinical safety, legal/privacy |
| Zespół | founder, frontend/UX, medical advisor, validation researcher |
| Milestone | dowód użyteczności briefu i historii pacjenta |
| Następna runda gdy | lekarze i pacjenci potwierdzają wartość workflow |

## Etap 2: pilot

| Obszar | Treść |
|---|---|
| Cel | zamknięty pilot z partnerem |
| Kwota orientacyjna | 0.8-1.8 mln PLN |
| Środki idą na | backend-ready contracts, privacy/security, pilot ops, support |
| Zespół | CTO/architect, frontend, backend, UX, legal/privacy, medical safety |
| Milestone | bezpieczny pilot bez realnych claimów klinicznych systemu |
| Następna runda gdy | pilot pokazuje adoption i brak harm signals |

## Etap 3: regional scale

| Obszar | Treść |
|---|---|
| Cel | kilka placówek / partnerów, procesy operacyjne |
| Kwota orientacyjna | 3-8 mln PLN |
| Środki idą na | product engineering, security, compliance, integrations prep |
| Zespół | pełny core product + GTM + ops |
| Milestone | powtarzalny workflow i partnerzy |
| Następna runda gdy | model płatności i wdrożenia jest powtarzalny |

## Etap 4: national-ready

| Obszar | Treść |
|---|---|
| Cel | gotowość do większych integracji i standardów |
| Kwota orientacyjna | 10-30 mln+ PLN |
| Środki idą na | skala, security, compliance, interoperability, support |
| Zespół | organizacja produktowo-regulacyjna |
| Milestone | enterprise/public-ready context layer |
| Następna runda gdy | są dowody skali i ścieżka partnerstw |

## Źródła finansowania

| Źródło | Kiedy pasuje |
|---|---|
| Bootstrapping | foundation i pierwsza walidacja |
| Granty | public interest, accessibility, open source, health data literacy |
| Angel / seed | MVP validation i pilot |
| VC | po dowodzie wedge i modelu skali |
| Partnerzy | pilot w placówce lub organizacji pacjenckiej |
| B2B / B2B2C | po potwierdzeniu workflow i buyer persona |

## Open source / public interest angle

Open source buduje zaufanie i umożliwia review safety, ale nie oznacza braku modelu biznesowego. Model może łączyć:

- otwarte standardy i demo;
- płatne wdrożenia;
- usługi walidacji i konfiguracji;
- komponenty enterprise;
- partnerstwa z placówkami.

## DoD

- Każdy etap ma cel, kwotę, zespół i milestone.
- Nie ma obietnicy skali bez dowodu.
- Funding jest powiązany z walidacją.

## DoE

- Pitch inwestorski pokazuje etap i next proof.
- Budżet nie zakłada dużego backendu przed walidacją.
- Founder potrafi powiedzieć, co kupuje pierwsze 150-400 tys. PLN.

## FoR

Review finansowania ma sprawdzać, czy pieniądze kupują redukcję ryzyka, nie tylko więcej funkcji.
